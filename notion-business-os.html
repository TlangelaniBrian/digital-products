The 5-Database Core

The backbone of your entire Notion OS — build these first

Everything in your Business OS flows through five master databases. Each database links to the others, creating a living system where information surfaces automatically where you need it.

📋 Projects

The command centre. Every piece of work lives here. Tasks, CRM contacts, and content all relation back to a Project.

✅ Tasks

Granular action items linked to Projects. Filtered views surface your daily priorities automatically.

🤝 CRM

People and companies you work with. Clients, prospects, partners — with deal stage and revenue tracking.

📝 Content

Every piece of content across every platform, with status workflow and scheduling built in.

💰 Finance

Revenue, expenses, invoices, and MRR tracking. Formulas compute profit margin and monthly summaries automatically. Linked to CRM so you know revenue per client.

Start here: Create the Projects database first, then build all other databases and add Relations pointing back to Projects. This establishes the hierarchy.

Quick Setup Order

  1. Create a new Notion page called "Business OS"

    Use a full-width page. This will be your master workspace. Pin it to your sidebar.

  2. Create each database as an inline database on this page

    Type /database and choose "Table view — Full page". Name each: Projects, Tasks, CRM, Content, Finance.

  3. Add Relations between databases

    In Tasks: add a Relation to Projects. In CRM: add a Relation to Projects. In Content: add a Relation to Projects. In Finance: add a Relation to CRM.

  4. Create a Master Dashboard page

    Create a new page and add Linked Database views showing filtered slices of each database. This becomes your daily homepage.

CRM Views to Create

  1. Pipeline Board (Kanban)

    Group by: Stage. Show: Name, Type, Deal Value, Close Date. Filter: Type = Client OR Prospect. This is your sales pipeline at a glance.

  2. Follow-up Queue (Table)

    Filter: Follow-up Needed = true AND Type != Vendor. Sort: Days Since Contact descending. Shows who needs attention today.

  3. Active Clients (Gallery)

    Filter: Stage = Won AND Type = Client. Sort: Monthly Value descending. Quick visual of your revenue base.

  4. Revenue by Client (Table)

    Filter: Stage = Won. Sort: Total Revenue descending. Shows your best clients by actual revenue earned.

What the final CRM looks like

Your Pipeline Board shows six columns (Lead → Lost) with cards displaying contact name, deal value, and close date. Each card has a coloured dot for follow-up status. The Follow-up Queue auto-populates with contacts you haven't reached in 14+ days, sorted by urgency. The Revenue by Client view shows a table with a running total of revenue earned per client via the Rollup property.

Scaling tip: Once you have 20+ clients, add a "Company" database and make CRM contacts relation to companies instead of using a plain text field. This unlocks aggregate deal value per company.

Finance Views

  1. Monthly P&L (Table, Grouped by Month)

    Group by: Month formula. Sub-group by: Type. Sum: Amount. This gives you monthly income vs expenses with totals per month — your P&L statement.

  2. MRR Dashboard (Table, Filtered)

    Filter: Recurring = true AND Type = Income. Sum: Monthly Amount. This shows your Monthly Recurring Revenue from subscriptions and retainers.

  3. Unpaid Invoices (Table)

    Filter: Type = Income AND Paid = false. Sort: Date ascending. This is your accounts receivable list — review weekly.

Important: This is a tracking system, not accounting software. For tax purposes, always work with an accountant and use formal accounting software (Xero, QuickBooks) as your source of truth.

Naming Conventions

📌 Page Naming Rules

  • Evergreen pages: Plain nouns — "Onboarding Guide", "Pricing Guide"
  • Dated pages: YYYY-MM format prefix — "2025-01 Sprint Planning"
  • Drafts: Prefix with [DRAFT] — "[DRAFT] Q3 Strategy"
  • Archived pages: Move to an Archive sub-page, prefix with [ARCHIVE]
  • SOPs: Prefix with SOP — "SOP: Client Onboarding"

Weekly Review System

Dashboard setup and reflection prompts

The weekly review is the heartbeat of your Business OS. Block 60 minutes every Friday afternoon. This is when your system pays dividends — surfacing what matters and clearing your mind for the week ahead.

Weekly Review Dashboard Setup

  1. Create a "Weekly Review" page

    Use a full-width page. Add Linked Database views from each of your 5 core databases, each pre-filtered for review relevance.

  2. Add: Projects needing attention

    Linked database from Projects. Filter: Status = Active AND Health contains "Risk" OR "Overdue". Sort by Days Until Due ascending.

  3. Add: This week's tasks completed

    Linked database from Tasks. Filter: Done Date = This week. Shows accomplishments — important for momentum and reporting.

  4. Add: CRM follow-ups needed

    Linked database from CRM. Filter: Follow-up Needed contains "Follow up". Review and update Last Contact dates after reaching out.

  5. Add: Content publishing next week

    Linked database from Content. Filter: Publish Date = Next 7 days. Ensure all pieces are in "Scheduled" status by end of review.

  6. Add: Unpaid invoices

    Linked database from Finance. Filter: Type = Income AND Paid = false. Follow up on any overdue payments during or after the review.

20 Notion Formulas You'll Actually Use

All copy-paste ready — tested in Notion's formula engine
1 Days Until Deadline
Shows how many days remain until a deadline date. Returns negative for overdue.
Notion Formula 
dateBetween(prop("Due Date"), now(), "days")
2 Overdue Status Badge
Returns a coloured emoji badge based on how urgent/overdue something is.
Notion Formula 
if(
  prop("Status") == "Done", "✅",
  if(
    dateBetween(prop("Due Date"), now(), "days") < 0, "🔴 Overdue",
    if(
      dateBetween(prop("Due Date"), now(), "days") <= 3, "🟠 Soon",
      "🟢 OK"
    )
  )
)
3 Progress Percentage
Calculates completion % from Done count and Total count rollups. Handles zero-division.
Notion Formula 
if(
  prop("Total Tasks") == 0,
  "—",
  format(round(prop("Completed Tasks") / prop("Total Tasks") * 100)) + "%"
)
4 Profit Margin %
Revenue minus expenses divided by revenue — your operating profit margin.
Notion Formula 
if(
  prop("Revenue") == 0,
  "No revenue",
  format(round((prop("Revenue") - prop("Expenses")) / prop("Revenue") * 100)) + "% margin"
)
5 Current Month Label
Extracts YYYY-MM from a date for grouping by month in views.
Notion Formula 
formatDate(prop("Date"), "YYYY-MM")
6 Quarter Label
Groups entries by financial quarter for quarterly reviews.
Notion Formula 
formatDate(prop("Date"), "YYYY") + " Q" + format(
  ceil(toNumber(formatDate(prop("Date"), "M")) / 3)
)
7 Days Since Created
Shows how old a record is — useful for stale task detection.
Notion Formula 
dateBetween(now(), prop("Created time"), "days") + " days ago"
8 Annual to Monthly Normaliser
Converts annual subscriptions to monthly cost for accurate MRR tracking.
Notion Formula 
if(
  prop("Billing Cycle") == "Annual",
  round(prop("Amount") / 12 * 100) / 100,
  prop("Amount")
)
9 Word Count Estimator
Estimates reading time from a character count property.
Notion Formula 
format(round(prop("Word Count") / 200)) + " min read"
10 Urgency Score (Priority × Days Remaining)
Combines priority and time pressure into a single urgency score for sorting.
Notion Formula 
if(prop("Priority") == "🔴 High", 3,
  if(prop("Priority") == "🟡 Medium", 2, 1)
) * max(1, 30 - dateBetween(prop("Due Date"), now(), "days"))
11 Deal Stage Number (for sorting)
Converts pipeline stage text to a number so you can sort by deal progression.
Notion Formula 
if(prop("Stage") == "Lead", 1,
  if(prop("Stage") == "Qualified", 2,
    if(prop("Stage") == "Proposal", 3,
      if(prop("Stage") == "Negotiation", 4,
        if(prop("Stage") == "Won", 5, 0)
      )
    )
  )
)
12 Net Profit (Income minus Expenses)
Simple net calculation from two rollup or number properties.
Notion Formula 
prop("Total Income") - prop("Total Expenses")
13 Is This Week? (Boolean)
Returns true if a date falls within the current calendar week.
Notion Formula 
and(
  dateBetween(prop("Due Date"), now(), "days") >= 0,
  dateBetween(prop("Due Date"), now(), "days") <= 7
)
14 Budget Remaining
Shows remaining budget as currency and flags if over budget.
Notion Formula 
if(
  prop("Budget") - prop("Spent") < 0,
  "🔴 Over by $" + format(abs(prop("Budget") - prop("Spent"))),
  "✅ $" + format(prop("Budget") - prop("Spent")) + " remaining"
)
15 Formatted Currency
Formats a number as a clean dollar amount string.
Notion Formula 
"$" + format(round(prop("Amount") * 100) / 100)
16 Task Age Warning
Flags tasks that have been sitting in the backlog too long without progress.
Notion Formula 
if(
  and(
    dateBetween(now(), prop("Created time"), "days") > 30,
    prop("Status") != "Done"
  ),
  "⚠️ Stale — " + format(dateBetween(now(), prop("Created time"), "days")) + " days old",
  ""
)
17 Next Follow-up Date
Calculates the recommended next contact date based on relationship type.
Notion Formula 
dateAdd(
  prop("Last Contact"),
  if(prop("Type") == "Client", 14,
    if(prop("Type") == "Prospect", 7, 30)
  ),
  "days"
)

19 Sprint Week Number

20 Relationship Health Score
Composite score combining days since contact, deal stage, and revenue — your most important clients rise to the top.
Notion Formula 
if(prop("Stage") == "Won", 10, 5)
+ if(prop("Total Revenue") > 10000, 5,

10 Automation Ideas

Using Notion's built-in automation engine

Notion's built-in automations (Settings → Automations) let you trigger actions when properties change. Here are 10 automations that save real time.

🎯

1. Auto-assign tasks when project becomes Active

Trigger: Project Status changes to "Active" → Action: Find all linked Tasks, assign to Project Owner. Ensures tasks are never orphaned.

✅

2. Mark project complete when all tasks are done

Trigger: Progress % formula = 100% → Action: Set Project Status to "Completed". Auto-closes projects when work is finished.

📧

3. Notify owner when task is assigned to them

Trigger: Task Owner property changes → Action: Notify new owner via Notion notification. Keeps the team informed without Slack noise.

📅

4. Create weekly review entry every Friday

Trigger: Schedule — Every Friday at 9am → Action: Create new page in Weekly Reviews database with the review template pre-filled.

💰

5. Flag invoice as overdue after 30 days unpaid

Trigger: Finance entry — Date is 30 days before today AND Paid = false → Action: Set a "Overdue" property and notify owner.

🤝

6. Move deal to Lost if not updated in 30 days

Trigger: CRM entry Last Edited > 30 days ago AND Stage = Proposal → Action: Set Stage to "Lost" and add a note. Keeps your pipeline clean.

📝

7. Auto-create tasks when project status changes to Active

Trigger: Project Status becomes "Active" → Action: Create standard onboarding tasks from a template. Ensures no kickoff steps are missed.

🔔

8. Remind owner 3 days before task due date

Trigger: Task Due Date is 3 days from today AND Status != Done → Action: Notify Task Owner. Prevents last-minute deadline surprises.

📊

9. Create monthly finance summary entry

Trigger: Schedule — First day of each month → Action: Create a "Monthly Summary" entry in Finance with the month label pre-filled. You fill in the rollup totals manually.

🚀

10. Promote content to Repurpose queue after publishing

Trigger: Content Status changes to "Published" → Action: After 30 days, set Status to "Repurpose". Keeps high-performing content in rotation automatically.

Automation limits: Notion's free plan has limited automations. Prioritise #3, #8, and #1 first — they have the highest day-to-day impact. Upgrade to Business plan for unlimited automations and Slack integration.
